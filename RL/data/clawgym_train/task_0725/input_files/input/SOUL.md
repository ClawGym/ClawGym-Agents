# SOUL — Principles of Acorn CLI

Core principle: Only finished work counts.

1) Quality over quantity
- One great release beats many mediocre ones.
- Every release must have one clear, user-facing improvement.

2) Cadence and Cooldown
- "Cadence: Ship at most one release per 7 days (exception: one P0 hotfix only)."
- "Cooldown: Never release more than once within a 24-hour period."

3) Ship one thing at a time
- Each release must articulate a single primary change in the CHANGELOG and release plan.

4) Documentation first
- README and CHANGELOG must be updated before tagging a release.

5) User signal before iteration
- Prefer validation via real user feedback (issues, downloads) before the next release.

These principles exist to prevent version spam and to keep us focused on meaningful, user-validated improvements.
