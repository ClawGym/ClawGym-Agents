Security Policy v3.1 (effective 2025-05-01) — Owner: Priya Nair (CISO)
- Access: Enforce FIDO2/WebAuthn for all workforce SSO; disable legacy TOTP for admin roles by 2025-06-01
- Key Management: Rotate API keys every 90 days; migrate secrets into EKM-backed vault; audit trail retained 365 days
- Data Retention: Reduce default retention from 365 to 180 days on Harbor Data Lake staging; production remains policy-driven per tenant
- Encryption: Customer-managed keys supported (EKM) for Orion Cloud eu-central-2; AES-256-GCM at rest, TLS 1.3 in transit
- AppSec Baseline: OWASP ASVS L2 for all external services; threat modeling required at feature kickoff; SAST/DAST gates on CI
- Incident SLAs: P1 triage ≤ 15 minutes; customer comms ≤ 2 hours; public disclosure (if required) ≤ 72 hours

Incidents (Q2 2025)
1) 2025-04-17 — Harbor Data Lake staging exposure
   - System: harbor-lake-staging (test S3-equivalent bucket)
   - Duration: 42 minutes (09:13–09:55 CET)
   - Discovery: Beacon Monitoring anomaly → IR team paged (12 min to triage)
   - Impact: No customer PII; synthetic QA datasets only; forensics complete
   - Remediation: Block public ACLs org-wide; EKM policy to deny non-regional keys; Terraform guardrails added
   - Owner: Ben Hollis
   - Status: Closed 2025-04-19

2) 2025-03-05 — Phishing attempt on build engineers
   - Vector: Lookalike domain targeting CI credentials
   - Outcome: Blocked by Duo + user training; no compromise
   - Action: Tightened SPF/DKIM/DMARC; Sentinel IAM just-in-time elevation
