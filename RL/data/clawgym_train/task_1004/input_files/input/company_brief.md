Product name: LevelMate — the AI Assistant for Game Development Teams

One‑liner:
LevelMate is a privacy‑preserving, multimodal coding copilot that understands game scenes, shaders, and build pipelines to help Unity, Unreal, and Godot teams ship faster with fewer regressions.

Target customers:
- Primary: Mid‑size game studios (15–150 employees) with 5–40 engineers, building PC/console titles or high‑fidelity mobile games.
- Secondary: Indie studios (3–10 devs) with live service or frequent content updates.
- Buyers: CTO/Head of Engineering, Technical Art Lead, Studio Producer.
- Users: Gameplay programmers, tools engineers, technical artists, QA automation engineers.

Core jobs‑to‑be‑done:
- Suggest and explain engine‑idiomatic code (C# for Unity, C++/Blueprints for Unreal, GDScript/C# for Godot).
- Diagnose graphics/shader issues using screenshots and build logs; propose fixes.
- Automate scene checks (missing references, broken prefabs/material slots).
- Orchestrate builds/tests, summarize diffs, and propose rollbacks for regressions.
- Help technical artists with shader graph edits and performance hints.

Key capabilities:
- Multimodal deep read: accept screenshots, short video captures, logs, and shader code.
- Tool integrations: Unity Editor (menu + context), Unreal Editor plugin, Godot plugin, CI (Cloud Build/TeamCity/GitHub Actions).
- Privacy & security: on‑prem or VPC deployment; project‑scoped retrieval; no training on customer code; SOC 2 roadmap.
- Governance: red‑flag checks for licensing conflicts (3rd‑party assets), unsafe code, and perf regressions.

Differentiation:
- Built for game workflows (scene/shader/asset awareness) vs. general code copilots (GitHub Copilot, Cody).
- Visual debugging and scene QA, not just code completion.
- Enterprise controls: PII/IP guardrails, deployment flexibility, audit logs.

Market assumptions:
- Studios are under pressure to ship frequent updates; build/test bottlenecks and graphics regressions are costly.
- Teams increasingly adopt AI for coding but need domain‑specific context and governance.

Pricing hypothesis (subject to validation):
- Starter (team up to 10 seats): $39/user/month SaaS (limited features, Unity focus).
- Pro (studio 25–100 seats): $79/user/month + usage (multimodal + CI + governance).
- Enterprise: Custom (on‑prem/VPC, SSO, audit, SLAs, dedicated support).

Goals and constraints:
- Beta readiness in 8 weeks; Unity integration first, Unreal beta by week 10, Godot research pilot only.
- Budget constraints favor 2–3 focused GTM channels.
- Quick‑win demand gen needed within 30 days; compounding channel for 6–12 months growth.
- Compliance: initial DPIA and data‑processing addendum by GA.

Competitive set:
- GitHub Copilot / Copilot Enterprise
- Sourcegraph Cody
- Replit Agents
- Unity Muse (niche overlap)
- Internal bespoke tools (scripts, editor extensions)

Success metrics (initial hypothesis):
- 20% faster bug‑fix cycle for graphics regressions in Unity projects.
- 30% reduction in shader‑related support tickets after 60 days.
- 15 active design partner studios by end of quarter.
