Minimum Necessary Placeholders for Patient Messages

The table below defines which placeholders are permitted in patient-facing messages for each consult type. Only the placeholders listed are allowed. All other PHI (e.g., full names, dates of birth, diagnosis codes, or complaints) should be excluded from messages.

Consult type Allowed placeholders (comma-separated)
general_med patient_first_name, visit_date, visit_time, provider_initial, meeting_link
behavioral_health patient_first_name, visit_date, visit_time, meeting_link
dermatology_photo_review patient_first_name, visit_date, meeting_link

Definitions: patient_first_name = given name only; provider_initial = first initial only; meeting_link = unique URL for video visit. Diagnosis codes, chief complaints, dates of birth, full names, phone numbers, and email addresses are not permitted in these categories.
