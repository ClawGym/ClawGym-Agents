Project: alpha

This guide explains how Palaia manages memory tiers for reliable search and fast recall. The system promotes fresh information while keeping older context available when needed. Tier rotation is handled by lightweight garbage collection routines that never block user queries. Daily operations focus on keeping the hot tier lean and responsive, while the warm tier preserves recent context. The cold tier stores long term knowledge for occasional lookups. When you run gc, tier rotation evaluates age and access frequency to decide movement between hot, warm, and cold tiers.

To ensure resilience, schedule daily gc and a weekly aggressive run that clears the cold tier when appropriate. Tier rotation protects performance as your memory grows. If your workflow needs privacy, remember that entries can be written with private scope, team scope, or public visibility. Private scope entries stay visible only to the writing agent. Team scope shares within the workspace. Public visibility is meant for export. Palaia also supports a configurable embedding chain so semantic search continues even when cloud providers are unavailable.

The embedding chain typically starts with a local model and falls back to bm25 keyword matching. This hybrid approach provides fast, offline friendly results while keeping behavior stable. When combined with careful tier rotation, your searches remain quick and dependable. Plan maintenance as a routine: daily gc for incremental rotation, and weekly aggressive gc to compact stale data and reduce index size. These practices keep the hot tier small, the warm tier useful, and the cold tier tidy, ensuring the agent’s memory remains crash safe, searchable, and predictable over time.
