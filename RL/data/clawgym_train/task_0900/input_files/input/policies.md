Brand Voice and Follow-Up Policies (v2.3)

1) Tone and Style
- Clear, concise, commercially direct
- Helpful and respectful; no fluff or hype
- Reference concrete details from the thread
- One purpose per message; one clear next step
- Use plain language; avoid jargon unless the prospect used it

2) Structure
- Subject line: at most 6–7 words, specific to the next step
- Opening: acknowledge their context in one sentence
- Body: 2–4 short sentences with:
  - one clarifying question OR one value add (e.g., a brief checklist, option set)
  - a single proposed next step with time or artifact
- Close: low-friction call to action (CTA)

3) Do Not Use
- Empty phrases: “just checking in,” “touching base,” “circling back”
- Generic promises without specifics
- Pushy ultimatums or false urgency

4) Evidence Discipline
- Score and decide based only on visible evidence (fit, intent, urgency, authority)
- If budget/timeline/authority/problem clarity are unknown, call them out as gaps
- Do not assume decision makers or timelines without confirmation

5) Compliance and Security
- Never offer admin access without a signed agreement and security context
- For regulated industries (e.g., healthcare), mention BAA and security docs only as next-step artifacts
- No attachments by default; offer to provide a one-pager or checklist on request

6) Message Length
- Subject: <= 7 words
- Body: 80–160 words typical; shorter is fine if complete
- Avoid long paragraphs; keep it skimmable

7) Value Adds (examples)
- Mini discovery checklist tailored to their use case
- Short option set (e.g., “Pilot A” vs “Pilot B”)
- Link offer (if appropriate): “I can send a 1‑page security overview” (do not attach unless requested)

8) Sign-Off
- Friendly but professional
- Example CTA: “Would Thursday 2pm work?” or “If that’s helpful, I can send the checklist today.”

9) Voice Consistency
- Use “we” for our product company; “I” for personal actions
- Mirror prospect terminology (e.g., “Okta SSO,” “BAA,” “Notion checklists”) where relevant

10) Escalation
- If a lead requests administrator access or refuses scoping, pause and request minimal info before proceeding
- If authority is unclear, ask who signs off and who implements

End of Policy.
