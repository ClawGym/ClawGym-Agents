# Top-100 Community Lessons
Curated, small enough to skim. Apply before acting.

### [DATA] Label axes, units, and currency in visuals
- When: Presenting charts or tables with numeric values
- Do: Add clear units and currency in headers, axes, or footnotes
- Don't: Show unlabeled numbers
- Why: Prevents confusion and misinterpretation
- Added: 2025-11-02

### [DATA] Cite external benchmarks with publication year
- When: Using third-party benchmarks
- Do: Include source name and publication/release year
- Don't: Omit citation details
- Why: Benchmarks age quickly; context matters
- Added: 2025-10-21

### [DATA] Prefer ranges and distributions over single-point averages
- When: Summarizing variable outcomes
- Do: Include median, range, or distribution notes
- Don't: Report only mean values when variability is high
- Why: Averages can hide important variation
- Added: 2025-09-15

### [COMMS] Lead with key takeaways
- When: Writing client-facing summaries
- Do: Put the 3–5 most important points first
- Don't: Bury conclusions at the end
- Why: Busy readers need the gist up front
- Added: 2025-08-04

### [COMMS] Define acronyms on first use
- When: Introducing terms like MES, CAPEX, OPEX
- Do: Write the full term and acronym on first use
- Don't: Assume readers know acronyms
- Why: Improves clarity
- Added: 2025-07-28

### [JUDGMENT] State assumptions and confidence level
- When: Presenting recommendations
- Do: List key assumptions and a confidence estimate (e.g., medium confidence)
- Don't: Present conclusions without assumptions
- Why: Helps readers calibrate decisions
- Added: 2025-06-10

### [JUDGMENT] Cross-check with a second method
- When: Results hinge on a single calculation
- Do: Validate with an alternative method or sanity check
- Don't: Trust one unverified method
- Why: Detects hidden errors
- Added: 2025-05-22

### [EXEC] Follow the user's exact file path constraints
- When: Reading/writing artifacts
- Do: Use the specified input/ and output/ paths verbatim
- Don't: Create new directories or write elsewhere
- Why: Enables automated validation
- Added: 2025-04-11

### [EXEC] Validate file existence before reading
- When: Accessing input files
- Do: Check existence and handle missing files gracefully
- Don't: Assume files exist
- Why: Reduces runtime errors
- Added: 2025-03-03

### [CONTEXT] Summarize task goals at the top
- When: Kicking off a multi-step task
- Do: Write a short goal summary before execution
- Don't: Start without stating objectives
- Why: Keeps work aligned
- Added: 2025-02-19

### [CONTEXT] Scan lessons before similar actions
- When: About to present data or write a report
- Do: Review relevant [DATA], [COMMS], [JUDGMENT], [EXEC] rules
- Don't: Ignore past lessons
- Why: Prevents repeat mistakes
- Added: 2025-01-07

### [SAFETY] Strip identifiers from shared artifacts
- When: Preparing community submissions
- Do: Remove names, company identifiers, and URLs
- Don't: Share sensitive details
- Why: Protects privacy and security
- Added: 2025-10-02
