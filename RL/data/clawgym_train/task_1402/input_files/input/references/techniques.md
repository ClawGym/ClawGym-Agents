# Interview Techniques Reference

Use these as prompts and guardrails during structured interviews.

1) Rapoport’s Rules (Steelmanning)
- Paraphrase the plan so clearly that the author says “Yes, exactly.”
- Name what’s strongest and most reasonable about it.
- Share any genuine learnings you gained.
- Only then critique, specifying where you’ll probe.

2) Jobs-to-be-Done (JTBD)
- What progress is the user trying to make?
- Pushes (pains of the current way)
- Pulls (attractions of the new way)
- Anxieties (what could go wrong)
- Habits (status quo anchors)

3) Five Whys
- Ask “Why is that important?” up to five layers to reach root causes.
- Convert the root cause into a concrete requirement or mitigation.

4) Inversion / Pre-Mortem
- Imagine it’s a year later and the project failed badly. What happened?
- Work backward to identify risks and counters.

5) Laddering
- Attribute → Consequence → Value
- Tie technical choices to business outcomes and values.

6) Scope and Triple Constraint
- If scope grows, ask: which gives—time or cost?
- Always capture explicit in-scope and out-of-scope items.

7) Branch Discipline
- One question at a time.
- Don’t proceed on “it depends”—clarify dependencies.
- Park unresolved items explicitly and note a plan to resolve.

8) Acceptance Criteria (Given/When/Then)
- Given (preconditions), When (action/trigger), Then (observable result)
- Avoid vague terms like “fast” or “user-friendly” without numbers.

9) Blind-Spot Checks
- Cognitive: anchoring, sunk cost, groupthink
- Organizational: zombie stakeholders, diffusion of responsibility
- Economic: perverse incentives, hidden costs

10) Synthesis Checklist
- Decisions, Open items (with owners/due), Risks with mitigations, Success metrics, Tradeoffs, Conflicts to reconcile
