Research Notes — Atlas Notes

Context
This document captures early insights on how users work with Obsidian-flavored Markdown and the minimal feature set needed for a cohesive project bundle.

## Key Findings
- Users prefer properties (YAML frontmatter) for consistent titles, tags, and aliases across notes.
- Wikilinks ([[Note Name]] and section links [[Note#Heading]]) are favored for internal navigation; they survive renames.
- Callouts (e.g., > [!info]) and embeds (![[Note#Section]]) improve scannability and reduce duplication.
- A single “hub” note increases discoverability; diagrams and checklists help align collaborators.
