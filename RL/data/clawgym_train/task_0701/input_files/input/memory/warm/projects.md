# Projects (WARM Tier)

## Privacy-First Research Digest — MVP
- Scope: 5–7 article summaries per issue, markdown-only output.
- Pipeline: manual curation → HTML-to-Markdown fallback → privacy filter → assemble digest.
- URL policy: Strip fragments, replace sensitive query values ([redacted]) and mask the rest ([masked]).
- Preferred writing tone: clear, concise, pragmatic.
- Daily check-in at 09:00 UTC — heartbeat.
- Success metrics draft: 40% open rate (first 100), 60% clickers, 10 replies in 30 days.
- Risks: Overbuilding, ignoring Content-Signal (reputation risk).
- Notes: OAuth and real-time ingestion are out of scope for v1.
- Updated: 2025-12-01

## Research Ops
- Triage inbox on Mondays; shortlist candidates for curation.
- Manual screenshotting is unnecessary; text-only is preferred.
- Do not store secrets in URLs.
