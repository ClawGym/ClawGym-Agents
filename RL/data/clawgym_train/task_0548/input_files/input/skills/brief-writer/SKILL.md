# Brief Writer

Turn scattered inputs into a tight, one-page brief with consistent structure and decision-ready highlights.

## Triggers
- You have messy notes, links, or transcripts and need a clean summary
- A stakeholder asks for an executive brief or one-pager
- You want consistent sections and phrasing across briefs

## Steps
1. Gather all inputs (notes, links, transcripts) and skim for key themes
2. Identify purpose, audience, and key decisions needed
3. Extract facts, metrics, and quotes; avoid speculation
4. Draft the brief using the chosen template and house style
5. Run a 3-part check: clarity, completeness, and bias
6. Deliver the brief plus a summary blurb for chat or email

## Output Templates
- Executive Brief (1-page, ~350–500 words)
  Title: <Project / Topic>
  Date: <YYYY-MM-DD>
  Owner: <Name/Team>

  1. Context
     - Why this matters: <one sentence>
     - Background: <1–2 short bullets>

  2. Key Findings
     - <bullet with metric or fact>
     - <bullet with quote or insight>

  3. Risks & Unknowns
     - <bullet>
     - <bullet>

  4. Recommendations
     - <action verb + outcome>
     - <action verb + outcome>

  5. Next Steps (DRI • When)
     - <step> — <DRI> • <date>
     - <step> — <DRI> • <date>

- Summary Blurb (for chat/email, <= 5 lines)
  <Topic>: <one-line takeaway>. Key: <3 bullets>. Rec: <1 line>. Next: <owner + date>.

## Edge Cases
- Conflicting sources: flag contradictions and cite both sides
- Sparse notes: call out gaps and propose follow-ups
- Sensitive data: anonymize names and redact secrets
- Outdated links: verify last updated; note if stale

## Rules
- Be direct and neutral; avoid hype
- Use short sentences and concrete metrics
- Attribute quotes and link to sources inline
- Do not invent data; if unknown, say so

## Why this skill
- Time saved: ~8–12 minutes per brief
- Frequency: 2–4 per week
- Value score: High
