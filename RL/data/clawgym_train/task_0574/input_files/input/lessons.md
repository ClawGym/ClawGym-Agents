# Lessons
Rules extracted from mistakes. Append after failing, scan before deciding.

### [DATA] Show sources for external stats

- When: Presenting external statistics in reports or summaries
- Do: Include the source and access date for each external stat
- Don't: Cite figures without attribution
- Why: Traceability builds trust and allows verification
- Added: 2026-04-10

### [COMMS] Explain methods before conclusions

- When: Presenting analytical findings to a client or stakeholder
- Do: Briefly describe the method and data origin before stating conclusions
- Don't: Jump straight to conclusions without contextualizing the approach
- Why: Context improves comprehension and reduces misinterpretation
- Added: 2026-04-11

### [EXEC] Confirm file paths before writing

- When: Creating or updating output files for a user task
- Do: Verify the target directory and filename match the user’s specification
- Don't: Write to default or assumed paths
- Why: Prevents misplaced artifacts and validation failures
- Added: 2026-04-12
