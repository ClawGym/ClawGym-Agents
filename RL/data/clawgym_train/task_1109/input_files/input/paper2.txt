Title: SwiftQueue: Latency-Aware Scheduling for RPC-Driven Microservices

Background
Tail latency in microservice-based systems degrades user experience and SLO compliance. Existing queueing policies often optimize mean throughput at the expense of tail behavior.

Methods
SwiftQueue combines deadline-aware prioritization with adaptive batching to reduce queueing delay for long-running RPCs while preserving throughput. We implement SwiftQueue in a production-grade RPC runtime and evaluate it against a fair-queueing baseline using three open microservice benchmarks (Bookstore, SocialGraph, MediaStream). Each experiment uses identical hardware and load profiles with n=30 runs per configuration. We report throughput, p99 and p99.9 latency, and memory overhead. Confidence intervals reflect run-to-run variability, and significance is assessed with two-sided tests where applicable.

Results
Across benchmarks, SwiftQueue increases steady-state throughput by 1.7x (Bookstore), 1.9x (SocialGraph), and 2.3x (MediaStream) over baseline. Aggregated across all runs, the throughput improvement is significant (paired t-test p=0.004). On SocialGraph, p99 latency decreased from 64 ms to 42 ms (37% reduction; 95% CI: [29%, 45%]); p99.9 latency decreased from 180 ms to 110 ms (39% reduction; 95% CI: [31%, 46%]). On Bookstore, p99 latency decreased from 58 ms to 40 ms (31% reduction; 95% CI: [24%, 38%]). On MediaStream, p99 latency decreased from 72 ms to 44 ms (39% reduction; 95% CI: [33%, 46%]). Memory overhead relative to baseline was 12% (95% CI: [9%, 15%]) due to per-queue metadata and adaptive batch control. Sensitivity analysis showed stable gains under bursty loads.

Discussion
SwiftQueue delivers consistent throughput gains and substantial tail latency reductions with modest memory overhead. The largest benefits occur in services with heavy-tailed RPC durations, and improvements persist under bursty traffic patterns without manual retuning.
