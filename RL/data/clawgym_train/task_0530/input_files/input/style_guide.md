Research Brief Style Guide (CTO Audience)

Purpose
- Provide a concise, decision-useful brief for a technical executive.
- Highlight what changed recently, why it matters, and what to watch next.

Structure
- Title: Start with a single H1 line beginning with “# ” (e.g., “# Research Brief: Fast-Evolving Topics”).
- Sections: Use one H2 per topic with the exact topic string from the input (e.g., “## Vector databases and hybrid search (HNSW, PQ, reranking)”).
- Under each topic, include 5–8 curated links. Each link is a single bullet item.

Bullet format (must match exactly)
- Start each item with “- [Title] (URL) — ” followed by a 1–2 sentence annotation.
- Keep titles concise and informative; avoid clickbait phrasing.
- An annotation should summarize the core takeaway and why it matters now.
- If a clear publication date is known, include it in the annotation in parentheses (e.g., “(Feb 2026)”); otherwise omit the date.

Tone and voice
- Crisp, neutral, and executive-friendly.
- Use precise, concrete phrasing; avoid marketing fluff and generic filler.
- Focus on implications for architecture, scalability, security, and roadmap risk.

Content selection
- Mix: At least one news article and at least one scientific/technical source per topic.
- Quality: Prefer primary sources (vendor docs, standards, arXiv, peer-reviewed venues, reputable news outlets).
- Diversity: Avoid clustering links from the same domain; aim for multiple distinct domains.
- Recency: Respect the specified time window; omit older material unless it is a definitive reference that directly supports a recent development.

Do and Don’t
- Do: Explain what the source adds beyond others, call out surprising results, benchmarks, or breaking changes.
- Do: Use consistent punctuation and sentence casing for titles.
- Do: Keep annotations to 1–2 sentences (roughly 25–50 words).
- Don’t: Include excluded domains from the provided list.
- Don’t: Over-index on press releases; balance with technical deep dives, benchmarks, or implementation notes.
- Don’t: Add images or non-text content.

Examples (pattern)
- [Concise Source Title] (https://example.com/path) — Key finding or update in 1–2 sentences with specific implications for engineering decisions (Mar 2026).

Final checks
- Ensure every section has 5–8 bullets and at least one “news” and one “science/technical” source.
- Ensure URLs are unique across the entire brief.
- Ensure all content is in English.
