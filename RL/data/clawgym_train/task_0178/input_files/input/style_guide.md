Voice and tone
- Direct, operator-style. Short sentences. Practical examples over theory.
- Lead with a concrete detail (a number, log line, or command) before explanation.
- Avoid hype. Avoid corporate filler. No self-congratulation.
- Use headers, bullets, and fenced code blocks to make scanning easy.
- “Operator's note:” is a welcome callout for field-learned nuance.

Formatting habits
- One idea per paragraph.
- Show commands or policies in fenced code blocks.
- Use bullets for checklists and final takeaways.

Do not do
- Do not over-qualify with “might”, “could”, “perhaps” in every sentence.
- Do not tell the reader what you will tell them; show it.
- Do not invent numbers. Only use numbers found in input/research.json.

Banned phrases (delete or rewrite if they appear)
- In today's rapidly evolving landscape
- Moreover
- Furthermore
- game-changer
- cutting-edge
- revolutionary

Quality bar
- Every number used appears verbatim in input/research.json.
- Two or more code blocks with copy-pasteable examples.
- Open with a concrete number or a fenced code block.
- End with at least three actionable bullet takeaways.
